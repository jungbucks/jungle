### 1.1 아키텍처 요약

- **순수 정적 SPA**: 빌드 도구·프레임워크·npm 런타임 의존성 **0**. 바닐라 ES 모듈 19개 + classic script 2개(data.js, boot.js) + CSS 1파일.
- **렌더링**: 해시 라우터(`#evalplan:grade` 식) → `render()`가 `#main.innerHTML` 전체 교체. 상태 = 모듈 스코프 변수 + localStorage.
- **성능 설계** (이 프로젝트의 자랑, 건드릴 때 주의): modulepreload 워터폴 제거 → SW 3단 캐시(HTML network-first / JS·CSS SWR / 나머지 cache-first) → index.html 정적 셸 = 첫 페인트가 곧 최종 레이아웃(규칙 10) → 폰트 서브셋 443KB.
- **보안**: CSP `script-src 'self'`(인라인 전면 금지), 이벤트는 전량 `data-on*` 위임(utils `registerActions`), Gemini 키는 헤더 전송.
- **규모**: JS 약 7,000줄(비공백) / CSS 약 1,500줄 / data.js 1,748줄(168KB). TODO/FIXME 마커 0개.

### 7.3 세특 생성기 — 보류 결정에 대한 재해석 (사용자 결정 존중)

사용자 판단(2026-07): "직접 LLM에 넣는 게 더 효율적" → **생성기는 보류. 타당함.** 자유 생성은 범용 LLM이 항상 이긴다.

- **만들 것**: "세특 프롬프트 빌더" — 담은 성취기준 + 평가 항목 + 관찰 키워드 몇 개를 입력받아 **훈령 준수 프롬프트**(금지 표현·글자수 규격 포함)를 조립해 클립보드 복사. 교사가 자기 ChatGPT/Claude/Gemini에 붙여넣음.
- **장점**: API 키 불필요(4원칙 2 통과), 학생 데이터가 사이트에 저장 안 됨(3 통과), 어떤 LLM과도 호환, 개발 비용 소.
- **만들지 말 것**: 사이트 내 직접 생성(API 종속·비용·개인정보 3중 위반).
