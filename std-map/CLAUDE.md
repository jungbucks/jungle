# 성취기준 은하 (std-map) — 유지보수 플레이북 (후임 모델용 진입점)

> 정보 교과 7과목 성취기준 137개를 옵시디언 그래프 뷰처럼 그리는 인터랙티브 지식 지도.
> 노드 = 성취기준(과목색 별) + **개념 태그(허브)**, 엣지 = 소속. 사용자 기획(2026-07-18).
> **기술 제약 해제 프로젝트**(사용자 명시) — D3 v7을 `lib/`에 동봉(오프라인 안전, CDN 아님). 다른 프로젝트의 바닐라 원칙을 여기 끌어오지 말 것.

## 구조

| 파일 | 역할 |
|---|---|
| `tools/build-data.mjs` | ★ 정글 `jungle/assets/data.js`(SUBJECTS)에서 성취기준 추출 + **TAG_RULES**(개념 사전, 이 프로젝트의 심장)로 태깅 → `data/graph-data.js` 생성 + 커버리지 리포트 |
| `data/graph-data.js` | 생성물(직접 수정 금지 — 규칙 고치고 재생성) |
| `index.html` | D3 force 그래프 전체(검색·과목 필터·이웃 강조·상세 패널·개념↔성취기준 왕복 탐색) |
| `lib/d3.v7.min.js` | 동봉 라이브러리 |
| `test/verify.mjs` | 데이터 정합 10건 / `test/probe.html` E2E 8종 |

## 절대 규칙

1. **원천은 정글 data.js** — 성취기준 문구를 여기서 고치지 말 것(정글에서 고치고 `node tools/build-data.mjs` 재생성).
2. **태그 규칙 수정 후 3종 세트**: 재생성 → 커버리지 리포트 통독(태그 0개 목록이 곧 작업 지시서) → `node test/verify.mjs`(커버리지 100% 검사 포함).
3. TAG_RULES는 서술어("탐색한다" 등) 오검출을 피해 **명사구 위주** — 광범위 동사 규칙을 넣으면 그래프가 한 덩어리로 뭉친다.
4. 정글 반영은 사용자 결정 대기(현재 독립 프로젝트). 반영 시 iframe 임베드 or 링크 카드 중 결정 필요.

## 검증

`node tools/build-data.mjs`(커버리지 100% 확인) → `node test/verify.mjs`(10건) → probe(file:// 가능, D3 동봉이라 오프라인 동작):
`msedge --headless=new --user-data-dir=<임시> --allow-file-access-from-files --virtual-time-budget=15000 --dump-dom "file:///.../test/probe.html"`
