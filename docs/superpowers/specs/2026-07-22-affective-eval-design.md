# 정의적 영역 평가 탭 (설계)

> 2026-07-22 · 대상 `assets/data.js`(신설 데이터) · 새 모듈 `assets/affective.js` · `assets/app.js`(서브탭) · `assets/style.css` · `tools/test.mjs`
> 수업/평가계획 탭에 서브탭 신설. 교사가 정의적 영역 평가계획을 짤 때 과목별 예시를 표로 가져다 쓴다.

## 1. 문제

교사는 평가계획서에 **정의적 영역 평가**를 넣어야 하는데 이게 은근히 귀찮다.
구성은 **정의적 영역 / 성취기준 / 평가내용** 3열인데, 매번 백지에서 만든다.
정글은 성취기준 "검색"에서 출발한 사이트라, 교육과정 내용체계의 세 범주
(지식·이해 / 과정·기능 / **가치·태도**) 중 성취기준만 담고 가치·태도가 없다.

## 2. 목표

과목을 고르면 → 그 과목의 영역(단원)별 정의적 영역 평가 예시가 **3열 표**로 뜨고 →
한글 표로 복사해 평가계획서에 바로 붙여넣는다.

## 3. 결정 사항 (2026-07-22 사용자 확정)

| 항목 | 결정 |
|---|---|
| 정의적 영역 열의 근거 | **교육과정 '가치·태도' 요소 기반** (공식 내용체계) |
| 행 단위 | **영역(단원)별** 1~2행 |
| 평가내용 열 | **평가방법(관찰·자기·동료) + 구체 예시** |
| 과목 범위 | 성취수준 보유 **6과목**(중정보·고정보·AI·데과·SW·**정보과학**). 프로그래밍만 보류 |
| 데이터 작성 | **Claude가 초안 작성 → 교사(사용자) 검수** |
| 데이터 구조 | **재사용 가능**하게 — 교육과정 탭 노출은 후속(이번 범위 밖) |

## 4. 데이터 구조 (재사용 가능)

`data.js`에 신설. 정의적 영역 탭 "전용"이 아니라 독립 구조로 둬, 나중에 교육과정
탭이 `values`(가치·태도)를 그대로 재사용할 수 있게 한다.

```js
const AFFECTIVE = {
  middle: [
    {
      domain: "디지털 문화",                          // SUBJECTS[middle].domains[].name 과 일치
      values: ["디지털 시민성", "정보보호 실천 의지"],   // 가치·태도 (교육과정 근거)
      codes: ["[9정05-02]", "[9정05-03]"],           // 관련 성취기준 (기존 데이터에 실재)
      assess: [
        { method: "관찰평가", desc: "모둠 토론에서 개인정보·저작권 보호 관점을 근거로 들어 주장하는지 관찰" },
        { method: "자기평가", desc: "디지털 예절·정보보호 실천 체크리스트 5문항 자기점검" }
      ]
    },
    // … 영역별로
  ],
  high: [ … ], ai: [ … ], ds: [ … ], sw: [ … ], cs: [ … ],
};
const AFFECTIVE_VERIFIED = false;   // 교사 검수 전 배포 게이트 (teacher-pay RULES_VERIFIED 방식)
```

- **6과목 27영역** (중정보 5 · 고정보 5 · AI 4 · 데과 4 · SW 5 · 정보과학 4).
- **`domain`은 `SUBJECTS[id].domains[].name`과 정확히 일치**해야 한다(정합성 테스트가 강제).
  방금 겪은 "소프트웨어와 생활 영역 키 오타" 재발 방지.
- **정보과학(`cs`)은 `getAchvKey`가 `_cs` 접미사를 붙인다** — 도메인명은 접미사 없이
  ("알고리즘"·"데이터 구조"·"프로그래밍"·"정보과학 프로젝트") 적는다. sw 오타와 같은 계열의 함정.
- **`codes`는 기존 성취기준에 실재하는 코드**여야 한다(정합성 테스트가 강제).
- **`values`·`assess`는 Claude 초안 → 교사 검수 대상.** 근거 없이 지어내지 않는다.

## 5. 근거 조사 방침 (구현 시)

`values`(가치·태도)는 두 근거를 **교차**해 작성한다:
1. **2022 개정 정보과 내용체계표의 '가치·태도' 내용요소**(교육부 고시 원문).
2. **성취수준(ACHIEVEMENTS) 서술의 정의적 표현** — 이미 "가치를 판단하고 내면화",
   "유연한 태도를 인식", "윤리적 문제의 해결 방안" 등이 들어 있어 교차 검증에 쓴다.

`assess`(평가내용)는 위 가치·태도를 평가하는 **방법 + 문항 예시**를 Claude가 작성한다.
관찰평가·자기평가·동료평가 중 영역 성격에 맞게. 이건 명백히 **교사 검수 대상**이다.

## 6. UI

### 6-1. 서브탭
`app.js`의 evalplan 서브탭 목록(현재 8개)에서 **'평가계획' 바로 다음**에 삽입:
`수업계획 · 평가계획 · [정의적 영역 평가] · 수행평가 루브릭 …`
서브탭 상태 키는 기존 `evalPlanSubtab` 패턴을 따른다(예: `'affective'`).

### 6-2. 본문
```
[과목 선택 ▾]                                    [전체 복사]

┌─────────────┬──────────────────┬─────────────────────────┐
│ 정의적 영역   │ 성취기준          │ 평가내용                 │
├─────────────┼──────────────────┼─────────────────────────┤
│ 디지털 시민성 │ [9정05-02]       │ ▸관찰평가 — …            │
│ 정보보호 실천 │ [9정05-03]       │ ▸자기평가 — …            │
├─────────────┼──────────────────┼─────────────────────────┤
│ …(영역별)    │ …                │ …                        │
└─────────────┴──────────────────┴─────────────────────────┘
```
- 과목 선택은 stdpicker의 과목 드롭다운과 같은 목록(`evalSubjects`)을 쓰되, **AFFECTIVE에
  데이터가 있는 과목만** 노출.
- **검수 전(`AFFECTIVE_VERIFIED=false`)**: 표 상단에 "검수 예정 — 예시 초안입니다" 배지.
- **복사**: 3열 그대로 한글 표(TSV + clipboardWriteHTML). 규칙 6(한글 붙여넣기) 준수.

## 7. 검증

`tools/test.mjs`에 정합성 테스트 추가(방금 만든 도메인 정합성과 같은 계열):
- 모든 `AFFECTIVE[id][].domain`이 `SUBJECTS[id]`의 도메인명에 실재하는가
- 모든 `AFFECTIVE[id][].codes`가 그 과목 성취기준에 실재하는가
- `values`·`assess`가 비어 있지 않은가

## 8. 배포 게이트 (검수 전 배포 금지)

- `AFFECTIVE_VERIFIED = false`로 시작. 탭에 "검수 예정" 배지.
- 교사(사용자)가 `values`·`assess` 전건 확인 후 `true`로. teacher-pay와 동일 패턴.
- verify는 데이터 "구조"만 검사(정합성). 문구의 교육적 타당성은 사람이 본다.

## 9. 범위 밖 (후속)

- **교육과정 탭에 가치·태도 노출** — 같은 `AFFECTIVE.values`를 재사용하는 작은 확장.
  이번엔 정의적 영역 탭에 집중(사용자 결정).
- **지식·이해 / 과정·기능** 범주 데이터화 — 내용체계 전체는 별건 대공사.
- **프로그래밍** — 전문교과라 성취수준(ABCDE)이 없어 정의적 근거의 절반이 빠진다.
  성취기준 텍스트만으로 넣는 건 근거가 얇아 이번엔 제외(사용자 결정 2026-07-22).
  (정보과학은 성취수준이 있어 이번 6과목에 포함됨.)
