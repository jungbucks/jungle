# 학기 단위 성취수준 — 선택 기반 문단 병합 (설계)

> 2026-07-21 · 대상 `assets/achv.js` · `assets/app.js` · `tools/test.mjs`
> 이 설계는 커밋 `b4aa155`(학기 단위 성취수준 모달)를 **교체**한다.

## 1. 문제

`b4aa155`가 만든 모달은 과목의 **전 단원 성취수준을 레벨별로 재편성해 나열**한다.
A수준 아래에 성취기준 9개가 9행으로 늘어서는 식이다.

교사가 필요한 것은 그게 아니다. 평가계획서에 들어가는 학기 단위 성취수준은
**A~E 각 레벨이 한 문단**이고, 그 문단은 **그 학기에 다루는 성취기준들의
해당 레벨 문장이 합쳐진 것**이다. 즉 두 가지가 빠져 있었다.

1. 어떤 성취기준을 학기에 다루는지 **고르는 단계**가 없다 (전 단원 강제).
2. 고른 것들이 **한 문단으로 합쳐지지 않는다** (성취기준별 개별 행).

## 2. 목표

교사가 성취기준을 체크하면 → A~E 5문단이 자동으로 만들어지고 → 한글 표에
그대로 붙여넣을 수 있다.

## 4. 흐름

기존 진입 버튼(`📊 학기 단위 성취수준`, `app.js:503`)은 유지하되,
누르면 **결과가 아니라 선택기**가 먼저 열린다.

```
📊 학기 단위 성취수준
        ↓
[stdpicker] 성취기준 선택            ← 기존 공용 모달 재사용, 새 UI 없음
        ↓ 확인 (codes)
[결과 모달] A~E 5행, 각 행이 한 문단
        ↓ ↩ 성취기준 다시 고르기      ← 선택으로 복귀(직전 선택 복원)
        ↓ 전체 복사
     한글 표 붙여넣기
```

`stdpicker.js`는 이미 과목 전환·"단원 모두 포함"·선택 개수 카운터·코드순 정렬을
갖추고 있다. `openStdPicker({ subjectIdx, preselected, selectAll:true, onConfirm })`을
호출만 하면 되고 **선택 UI는 새로 만들지 않는다**.

## 5. 병합 규칙

### 5-1. 두 모드를 토글로 지원

결과 모달 상단 세그먼트 토글. 기본값 `join`.

| 모드 | 결과 |
|---|---|
| `join` **연결어미로 잇기**(기본) | `~구조화할 수 있으며, ~내면화할 수 있고, ~내면화할 수 있다.` |
| `raw` **원문 그대로** | `~구조화할 수 있다. ~내면화할 수 있다. ~내면화할 수 있다.` |

- 토글 변경 시 화면 즉시 재렌더.
- **`전체 복사`는 현재 보이는 모드 그대로 복사한다** (보이는 것 = 복사되는 것).

### 5-2. 어미 변환 (`join` 모드)

마지막 문장은 원문 그대로 두고, 앞 문장들만 종결어미를 연결어미로 바꾼다.
단조로움을 피해 `~으며,`와 `~고,`를 번갈아 쓴다.

| 원문 어미 | 변환 (비종결) |
|---|---|
| `...있다.` | `...있으며,` / `...있고,` |
| `...한다.` | `...하며,` / `...하고,` |

위 두 규칙에 걸리지 않는 문장은 **변환하지 않고 원문 그대로 이어 붙인다**
(데이터가 늘어 새 어미가 들어와도 문장이 깨지지 않게 하는 안전판).

### 5-3. 순수 함수로 분리

```js
mergeLevelTexts(texts /* string[] */, mode /* 'join' | 'raw' */) -> string
```

DOM을 모른다. 문자열 수술이라 회귀가 조용히 나기 쉬우므로
**`tools/test.mjs`에 단위 테스트를 함께 추가**한다(CLAUDE.md 규칙 1의 취지).

테스트 케이스:
- `join`: `있다.` 2개 연결 / `한다.` 연결 / `있다.`+`한다.` 혼합
- `join`: 어미 교대(`으며` → `고` → 종결) 확인
- `join`: 규칙 밖 어미는 원문 보존
- `join`/`raw`: 1개만 선택 시 원문과 동일(연결 없음)
- 빈 배열 → 빈 문자열
- `raw`: 공백 1칸으로 연결

## 6. 출력

- **화면**: A~E 행마다 병합 문단 + 그 아래 작은 출처 코드 배지(`[9정03-01] [9정03-02] …`)
- **복사**: 레벨·문단 2열만. **코드 없음.**
- 성취기준 순서는 stdpicker가 코드순 정렬해 넘긴 순서를 따른다.

## 7. 상태

모듈 최상단 변수 2개. **localStorage를 쓰지 않는다.**

```js
let _semSelected = [];      // 선택된 성취기준 코드
let _semMode = 'join';      // 병합 모드
```

- 새로고침하면 초기화된다. 학기 단위 성취수준은 한 번 뽑아 붙여넣고 끝나는
  작업이라, 백업 키·복원 경로를 늘려 얻을 값이 없다.
- **과목을 바꾸면 `_semSelected`를 비운다.** 중학교 정보 코드를 든 채
  고등학교 정보 화면에 남으면 조용히 틀린 결과가 나온다 — 이 초기화만은 명시적으로 한다.
