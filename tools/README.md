# 정글 배포 전 검증 (verify)

GitHub에 push하기 **전에** 반드시 실행. 이번에 겪은 "사이트 전체 백지" 같은 사고를 배포 전에 잡는다.

## 실행

```powershell
cd C:\Users\정벅\jungle
node tools/verify.mjs
```

- **종료코드 0** = 모든 검증 통과 → 배포 안전
- **종료코드 1** = 문제 발견 → 배포 중단 (어느 파일이 왜 깨졌는지 지목됨)

## 검사 항목

| # | 검사 | 왜 필요한가 |
|---|------|-------------|
| 1 | **import/export 정합성** | import한 이름이 실제 export 안 됐으면 브라우저에서 모듈 그래프 전체가 죽어 화면이 백지가 된다. `node --check`로는 못 잡는다. |
| 2 | **파일별 문법** | 깨진 파일을 **정확히 지목**한다. |
| 3 | **전체 그래프 로드 + init 실행** | `node --check`가 통과해도 실제 로드 시 터지는 런타임 초기화 오류를 잡는다. (렌더 함수 손상, 정규식 치환 사고 등) |
| 4 | **SW 프리캐시 파일 존재** | `sw.js` ASSETS 목록의 파일이 하나라도 없으면 `cache.addAll`이 통째로 실패해 서비스워커가 안 깔린다. 파일 추가/이름변경 후 필수. |

## 권장 워크플로

1. 코드 수정
2. `node tools/verify.mjs`
3. ✅ 통과하면 → `sw.js`의 `CACHE` 버전 올리고 → commit & push
4. ❌ 실패하면 → 지목된 파일 수정 후 2번부터 반복

## 네거티브 테스트 (검증기 자체가 살아있는지 확인)

아무 `assets/*.js` 끝에 `}` 하나를 임시로 넣고 실행 → `[2]`와 `[3]`에서 잡히고 종료코드 1이 나오면 정상. 확인 후 반드시 원복.
