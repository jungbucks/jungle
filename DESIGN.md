## 2-1. 고정 토큰 (변경 금지 — 새 hex를 발명하지 말 것)

### 정글 (assets/style.css `:root`에서 추출 — 이 값이 진실)
- **그레이 스케일**: `--g50 #F9FAFB … --g900 #111827`(다크모드는 통째 재매핑, 컴포넌트가 g토큰만 쓰면 자동 대응).
- **accent(브랜드 파랑)**: `--accent #2563EB` / light `#EFF6FF` / dark `#1E40AF` / line `#BFDBFE`. 과목 목록 페이지만 `setAccent()`로 3종 덮어쓰고 `render()`의 `resetAccent()`가 매번 복원(이 쌍 깨면 색 오염 버그).
- **브랜드**: `--brand #1B4D3E`(딥그린, 다크에서 `#34D399`).
- **상단 메뉴 5계열 체계 (2026-07-09 사용자 확정 — 이 규칙이 메뉴·눈썹 색의 진실)**:
  ① 교육과정(+시뮬레이터) = `--accent` 파랑 ② 성취기준 = 과목 고유색(**유일한 다색 허용 구역**) ③ 수업/평가계획 8도구 = `--plan` 보라 ④ 교과서 = `--book` 초록 ⑤ 자료실 3종(수업 도구·수업 사이트·앱스토어) = `--brand` 딥그린(`--brand-soft/-dark` 토큰 신설, 다크 재매핑 있음. data.js에서 `var(--brand)` 문자열로 참조 — hex 아님).
  **눈썹(페이지 맨 위 배지) = 소속 메뉴 색.** 도구 가족색(계산기=teal)은 페이지 본문 포인트로만 쓴다.
- **`--ovr` 인디고는 교육과정 대표색에서 퇴역**(2026-07-09). 잔여 사용처는 보조 장식(textbook 고시외 헤더, resources 카테고리 사이클)뿐 — 새 UI에 쓰지 말 것. dsai의 DS측 인코딩(`--dac-accent`=teal)은 의미 구분용이라 유지.
- **시맨틱**: ok `#10B981` / danger `#EF4444` / warn `#D97706`.
- **타이포 토큰**: `--fs-caption 13px / --fs-body 15px / --fs-body-lg 16px / --fs-title 18px / --fs-heading 21px / --fs-display clamp(24,3vw,28)`.

### 코드 낙하 (Gen Z Tech — 이 5색 고정)
배경 `#F8FAFC` / 본문 `#1E1B4B` / 포인트 코랄 `#F43F5E` / 포인트 블루 `#3B82F6` / 코드블록 배경 `#0F172A`. 그라디언트는 코랄↔블루만.

### 정보카츄 (개인 아카이브)
배경 `slate-900`(#0F172A 계열) + 포인트 `yellow-400`(#FACC15). 다크 위 옐로우 1포인트.

### 컴포넌트 관례 (정글 코드에서 추출 — 구체 수치)
- **transition**: 색·경계 `.15s`, 서랍/모달 슬라이드 `.22~.26s cubic-bezier(.4,0,.2,1)`. 그 이상 금지.
- **border-radius**: 입력·버튼 `8~10px`, 카드/셀 `12~14px`, 알약/배지 `20px+`. 임의 값 말고 이 범위.
- **그림자**: 헤더/카드 `0 1px 3px rgba(0,0,0,.08)`(은은). 강조 모달만 더 진하게. **모든 요소에 그림자 금지**.
- **hover**: 색 변화 위주. 이동은 최대 `translateY(-2~3px)`. 확대·회전·글로우 남용 금지.

## 2-2. 감각 승계 (그대로 따라 하면 됨)

- **타이포 스케일**: 본문 15~16px/행간 1.6~1.75. 제목은 본문의 약 1.4배(21px)·굵기 700~800. 캡션 13px(하한). **13px 미만 금지**. 위계는 크기+굵기로, 색은 마지막.
- **여백 시스템**: **8px 그리드 하나로 통일**(4·8·12·16·20·24…). 3단계: 요소 간 8px, 카드 내부 패딩 16~20px, 섹션 간 24~32px. 임의 7px·13px 여백 금지.
- **색 사용 비율**: 화면의 ~90%는 배경/그레이, ~8% 본문 잉크, **포인트(accent/도구색)는 화면당 3곳 이내**. 포인트 남용이 하위 모델 1등 실수 — CTA·활성 탭·현재 상태 정도로만.
- **인터랙션 강도**: duration 150ms 기본(최대 260ms), 이동 2~3px. "튀는" 애니메이션·자동재생 모션·파티클 금지.
- **위계 우선순위**: ① 크기 → ② 굵기 → ③ 여백 → ④ 색. **색으로 위계 만들지 말 것**(색은 의미 전달용, 크기·굵기로 먼저).

## 2-2b. 안티패턴 블랙리스트 (습관적으로 만드는 뻔한 실패 — 금지)

- ❌ 보라→파랑 그라디언트 히어로 배너(대형 gradient text 포함). 정글은 단색 브랜드.
- ❌ 의미 없는 이모지 아이콘 남발(버튼마다 이모지). 아이콘은 stroke SVG, 이모지는 의미 있을 때만.
- ❌ 무조건 3열 카드 그리드 강박 — 내용 수에 맞춰 `auto-fill minmax()`로.
- ❌ 모든 카드에 진한 그림자·모든 요소 hover 확대.
- ❌ 새 hex·새 폰트 크기·새 radius 발명(토큰·범위 안에서).
- ❌ 화면당 포인트 컬러 4곳 이상, 경고/성공을 색만으로 표시(§DECISIONS 접근성).
- ❌ 카드 안에 카드 안에 카드(중첩 3단+), 과한 테두리+배경+그림자 동시.

## 2-2c. 좋은 예 / 나쁜 예 (추상 원칙보다 이 쌍을 모방)

**카드**
```html
<!-- GOOD: 토큰·은은한 그림자·라운드 14 -->
<div style="background:#fff;border:1px solid var(--g200);border-radius:14px;
     padding:20px;box-shadow:0 1px 3px rgba(0,0,0,.08)">…</div>
<!-- BAD: 하드코딩 hex·과한 그림자·그라디언트 -->
<div style="background:linear-gradient(135deg,#7C3AED,#3B82F6);border-radius:24px;
     box-shadow:0 20px 60px rgba(0,0,0,.4);color:#fff">…</div>
```
**버튼**
```html
<!-- GOOD: 위임 핸들러·토큰·transition .15s -->
<button class="pbtn pri" data-onclick="tool:go">계산하기</button>
<!-- BAD: 인라인 onclick(CSP 차단)·이모지 범벅·하드코딩 -->
<button onclick="go()" style="background:#7C3AED">🚀✨ 계산하기 ✨🚀</button>
```
**모달**
```html
<!-- GOOD: 포커스 트랩·Escape·aria -->
<div role="dialog" aria-modal="true"> … utils.trapFocus 사용 … </div>
<!-- BAD: 스크롤 방치·포커스 미이동·닫기 키보드 불가 -->
<div class="popup" style="position:fixed"> … </div>
```

## 2-3. 판단 위임 규칙 (이 문서에 없는 결정)

1. **기존 화면에서 가장 유사한 컴포넌트를 찾아 그대로 모방**(창작 금지). 예: 새 도구 카드가 필요하면 gradecalc/rubric의 카드 구조를 복사해 변형.
2. 토큰·범위로 커버 안 되는 신규 시각 요소 → 먼저 `:root`에 새 토큰이 정말 필요한지 재검토, 대개는 기존 도구색(plan/book/teal/ovr) 재사용으로 해결.
3. 그래도 애매하면 **사용자에게 시안 2개를 텍스트로 요약해 질문**(예: "A안: teal 단색 배지 / B안: 아이콘+라벨 칩 — 어느 쪽?"). 이미지 없이 말로 대비.
