# DECISIONS.md — 애매할 때 "Fable이라면 이렇게 결정한다"

> CLAUDE.md(구조·규칙)·DESIGN.md(디자인)로 안 풀리는 판단만 여기서. 전부 검증 가능한 기준.

## 1. 새 파일인가, 기존 파일 확장인가

**기본값은 기존 파일 확장.** 아래 중 **2개 이상** 해당할 때만 새 파일:
- 독립 도구/화면이다(자체 render 함수를 export하고 라우터가 디스패치).
- 200줄 이상이 될 것이고, 기존 파일과 상태를 거의 공유하지 않는다.
- 다른 도구가 import할 공용 로직이다(→ 이건 utils.js/state.js로).

**새 파일을 만들면 반드시**: ① app.js 라우팅에 연결 ② index.html modulepreload + sw.js ASSETS 등록(규칙 3) ③ 매핑 표(CLAUDE.md)에 한 줄 추가.
**하지 말 것**: 기존 파일을 "정리한다"며 쪼개거나 합치기. app.js·evalplan.js는 상태 결합이 깊어 분해 금지(순환 import 지뢰). 도구 로직을 데이터 파일(data.js)에 섞지 말 것.

## 2. localStorage vs sessionStorage vs 비영속(메모리)

정글은 **공용 교무실 PC** 전제. 그 위에서:
- **비영속(메모리 변수)**: 학생 개인정보(이름·번호·점수). gradecalc가 선례 — 탭 닫으면 소멸, beforeunload 경고. **학생 행을 저장하는 코드를 넣지 말 것.**
- **localStorage**: 교사 본인 설정·작업물(반영비율·루브릭·계획 등, 개인 식별 정보 아님). 기기에 남아도 되는 것.
- **sessionStorage**: "이 탭에서만·새로고침엔 유지·닫으면 소멸"이 맞는 것(예: 페이지 접근 관문 통과 플래그).
- 판단 한 줄: **개인정보면 비영속, 교사 설정이면 localStorage, 세션 한정이면 sessionStorage.**
- 새 localStorage 키를 만들면 backup.js `BACKUP_KEYS`도 갱신(개인정보·API키는 제외 유지).

## 3. API 키 다루기 (공유 PC 대비)

- 저장은 **localStorage 평문**(정적 사이트라 서버 금고 없음)이되, 반드시 **① 공용 PC 경고 문구 ② '저장된 키 삭제' 버튼** 제공(gemini.js 선례).
- 전송은 **URL 쿼리 금지 → 헤더**(`x-goog-api-key`). 로그·리퍼러 유출 방지.
- 키는 **백업/복원 대상에서 제외**(BACKUP_KEYS에 넣지 말 것).
- BYOK 원칙: 운영자(사용자)가 API 비용을 대지 않는다. "키 없이 되게 해달라"는 요청엔 응하지 말 것(프록시=과금 시작).

## 4. 접근성 최소선 (타협 불가)

- **색상만으로 정보 전달 금지**: 정답/오답·상태·경고는 색 + (아이콘/모양/텍스트) 병행. 이상치 빨강이면 텍스트·아이콘도 같이.
- **키보드**: 모든 상호작용이 키보드로 가능. 모달은 포커스 트랩(utils.trapFocus)+Escape+열 때 포커스 이동. 포커스 링 제거 금지.
- **대비**: 본문 텍스트 4.5:1↑, UI 그래픽 3:1↑. 13px 미만 금지(규칙 7).
- **모션**: `prefers-reduced-motion`에서 큰 애니메이션 대체. 자동 재생 모션 지양.
- **aria**: 동적 결과 영역 `aria-live`, 표 입력에 라벨, 모달 `role/aria-modal`.

## 5. 복사/내보내기 (정글 핵심 UX)

- 목적지는 **한글(HWP)·워드**. 표는 **탭 구분(TSV)**, 빈 줄 제거, 서식이 필요하면 스타일 인라인한 HTML(clipboardWriteHTML). 이 HTML엔 CSS 변수 안 먹으므로 **리터럴 hex**(규칙 5 예외).
- 파일 내보내기는 외부 의존 없이 **Blob**로 생성(CSV는 UTF-8 BOM으로 엑셀 한글 호환).

## 6. "이럴 땐 반드시 사용자에게 물어라" (에스컬레이션)

- **되돌리기 어려운/외부로 나가는 작업**: 대규모 순서·구조 재배치, 파일 삭제·이름변경, 실제 배포(push/업로드), 외부 서비스 전송.
- **도메인 정확성이 걸린 판단**: 성취기준/교육과정/성적 산출 규정(예: 중간석차 vs RANK.EQ, 등급 경계) — 규정 실물은 교사만 안다.
- **요청과 코드/규칙이 충돌**: 예) "F12 차단" 같은 원리적으로 불가·부작용 있는 요청 → 되는 범위를 정직하게 제시하고 확인.
- **명세와 실제가 어긋남**: 준 값이 코드 사실과 다를 때(예: 해시가 원문과 불일치) → 실제를 우선하되 사용자에게 알림.
- **디자인이 DESIGN.md로 안 풀림**: 시안 2개를 텍스트로 요약해 질문(§DESIGN 2-3).
- 반대로 **묻지 말 것**: 규칙·매핑으로 답이 정해진 것, 되돌리기 쉬운 기본값 선택 → 그냥 진행하고 결과 보고.
